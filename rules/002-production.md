# Правило 002. О производстве

> Уровень: Правила (51%+ голосовавших, 3 дня cooling, x1 залог)
> Статус: драфт

---

## Раздел 1. Общие принципы

1.1. Производство — превращение одних ресурсов в другие по рецепту.
1.2. Ресурсы списываются при подаче заявки, результат начисляется
     после обработки.
1.3. Каждый гражданин может подать до 3 заявок на производство в день.
1.5. Добыча сырья (правило 001, раздел 3) не считается производством
     и не входит в лимит.

## Раздел 2. Рецепты

### Уровень 1: Сырьё → Материалы

| Рецепт | Вход | Выход |
|--------|------|-------|
| Распил | 3 wood | 2 plank |
| Обжиг кирпича | 4 clay | 3 brick |
| Плавка | 3 ore | 1 metal |
| Помол | 4 grain | 3 flour |

### Уровень 2: Материалы → Продукция

| Рецепт | Вход | Выход |
|--------|------|-------|
| Ковка инструмента | 2 metal + 1 plank | 1 tool (прочность 10) |
| Выпечка | 2 flour | 3 bread |
| Сборка механизма | 3 metal + 2 plank | 1 mechanism |

### Уровень 3: Комплексная продукция

| Рецепт | Вход | Выход |
|--------|------|-------|
| Строительство | 10 brick + 5 plank + 2 mechanism | 1 building |

## Раздел 3. Свойства продукции

### tool (Инструмент)
- Удваивает добычу сырья (правило 001, раздел 3.3).
- Прочность: 10. Теряет 1 за каждое использование.
- При прочности 0 — уничтожается.

### bread (Хлеб)
- Торговый товар. Не имеет механического эффекта в текущих правилах.
- Может получить эффект через будущие PR.

### mechanism (Механизм)
- Компонент для строительства.
- Может получить самостоятельный эффект через будущие PR.

### building (Строение)
- Увеличивает лимит производственных заявок владельца на +1.
- Эффект суммируется: 2 building = лимит 5 заявок в день.
- Не изнашивается.

## Раздел 4. Производственные цепочки (справочно)

```
wood ──→ plank ──┬──→ tool (+ metal)
                 ├──→ mechanism (+ metal)
                 └──→ building (+ brick + mechanism)

stone ──→ (прямое использование в будущих рецептах)

grain ──→ flour ──→ bread

ore ──→ metal ──┬──→ tool (+ plank)
                ├──→ mechanism (+ plank)
                └──→ (будущие рецепты)

clay ──→ brick ──→ building (+ plank + mechanism)
```

## Раздел 5. Добавление рецептов

5.1. Новые рецепты добавляются через PR на уровне "Правила".
5.2. PR должен содержать: рецепт, балансировку, use case, тесты.
5.3. Рецепт не должен создавать ресурсы из ничего (вход обязателен).
5.4. Рецепт не должен быть строго выгоднее существующего
     (обоснование баланса обязательно в PR).
