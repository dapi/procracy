# Конституция PRocracy

> Версия 0.1 — Учредительный драфт
> Статус: черновик, требует ратификации первыми гражданами

---

## Преамбула

PRocracy — самоуправляемая экономическая игра с открытым исходным кодом.
Игроки взаимодействуют с миром через агентов (MCP/API), создают экономику,
торгуют, производят и совместно определяют правила по которым эта экономика работает.

Настоящая Конституция определяет неизменяемые принципы, структуру управления
и механизмы изменения правил.

---

## Раздел I. Неизменяемые принципы

Данные принципы не могут быть изменены, отменены или обойдены никаким голосованием,
PR или решением.

### Статья 1. Открытость
1.1. Исходный код сервера, правила и состояние мира — публичны.
1.2. Любой может читать код, правила и историю всех изменений.
1.3. Алгоритмы эмиссии, рынка и производства не могут быть скрытыми.

### Статья 2. Право на участие
2.1. Любой человек может подать заявление на гражданство через PR по установленной форме.
2.2. Никто не может быть лишён гражданства без нарушения Конституции.
2.3. Банкротство не лишает гражданства и права голоса.

### Статья 3. Собственность
3.1. Ресурсы, принадлежащие игроку, не могут быть конфискованы без его согласия,
     кроме случаев предусмотренных Законами.
3.2. Каждая транзакция записывается и доступна для аудита.

### Статья 4. Самоуправление
4.1. Правила игры определяются исключительно голосованием граждан.
4.2. Ни один игрок, группа или внешняя сторона не имеет привилегированных прав
     на изменение правил в обход голосования.
4.3. Администратор сервера обеспечивает инфраструктуру, но не имеет
     игровых привилегий.

## Раздел II. Иерархия нормативных актов

Все изменения в игре классифицируются по уровню воздействия.
Каждый уровень требует разного порога для принятия.

### Статья 5. Уровни изменений

| Уровень | Что регулирует | Порог голосов | Cooling period | Залог |
|---------|---------------|---------------|----------------|-------|
| Конституция | Фундаментальные принципы и права | 75%+ граждан | 7 дней | x3 |
| Законы | Базовые механики (эмиссия, рынок, governance) | 60%+ голосовавших | 5 дней | x2 |
| Правила | Параметры, балансировка, новые механики | 51%+ голосовавших | 3 дня | x1 |
| Код | Реализация, баги, оптимизация (без изменения правил) | 2+ approve | 1 день | — |

5.1. PR, затрагивающий несколько уровней, проходит по наивысшему порогу.
5.2. Уровень PR определяется автоматически через CI на основе изменённых файлов.
5.3. Спорные случаи классификации решаются голосованием по порогу уровня "Правила".

### Статья 6. Структура репозитория по уровням

```
constitution/    → уровень "Конституция"
laws/            → уровень "Законы"
rules/           → уровень "Правила"
server/          → уровень "Код"
citizenship/     → заявления на гражданство
```

## Раздел III. Гражданство

### Статья 7. Получение гражданства
7.1. Для получения гражданства необходимо подать PR с заявлением
     в директорию `citizenship/` по установленной форме.
7.2. Заявление принимается на уровне "Код" (2+ approve, 1 день cooling).
7.3. После принятия гражданин получает:
     - Право голоса на уровне "Правила" и "Код" — немедленно.
     - Право голоса на уровне "Законы" — после 7 дней гражданства.
     - Право голоса на уровне "Конституция" — после 30 дней гражданства.
7.4. Один GitHub-аккаунт = один гражданин.

### Статья 8. Стартовые условия
8.1. Новый гражданин получает стартовый набор ресурсов, определённый в Правилах.
8.2. Размер стартового набора может меняться голосованием на уровне "Правила".

---

## Раздел IV. Голосование

### Статья 9. Процедура голосования
9.1. Голосование проходит на GitHub через реакции на PR.
9.2. :thumbsup: (палец вверх) — голос "за".
9.3. :thumbsdown: (палец вниз) — голос "против".
9.4. Автор PR не может голосовать за собственный PR. Его голос не учитывается.
9.5. Каждый гражданин имеет один голос на каждый PR.

### Статья 10. Срок жизни PR
10.1. PR на уровне "Конституция" — 14 дней на голосование.
10.2. PR на уровне "Законы" — 10 дней.
10.3. PR на уровне "Правила" — 7 дней.
10.4. PR на уровне "Код" — 5 дней.
10.5. Если за отведённый срок PR не набрал необходимых голосов — он закрывается.
10.6. Автор может повторно подать PR не ранее чем через cooling period.

### Статья 11. Залог
11.1. Для подачи PR уровня "Правила" и выше необходимо внести залог
      в игровой валюте.
11.2. Залог блокируется на время голосования.
11.3. Если PR принят — залог возвращается + вознаграждение из казны.
11.4. Если PR отклонён — залог сгорает в казну.
11.5. Размеры залогов определяются в Законах.

---

## Раздел V. Экономика

### Статья 12. Ресурсы
12.1. В мире существуют конечные и возобновляемые ресурсы.
12.2. Перечень ресурсов, их тип и параметры определяются в Правилах.
12.3. Добыча конечного ресурса уменьшает его мировой запас.
12.4. Возобновляемые ресурсы восстанавливаются по формулам, определённым в Правилах.

### Статья 13. Эмиссия
13.1. Эмиссия системной валюты происходит только по правилам,
      определённым в Законах.
13.2. Любое изменение правил эмиссии требует голосования на уровне "Законы".
13.3. Все акты эмиссии записываются в публичный журнал.

### Статья 14. Свободный рынок
14.1. Игроки свободны в установлении цен, заключении сделок
      и создании контрактов.
14.2. Игроки могут создавать собственные финансовые инструменты
      (валюты, контракты, деривативы) без одобрения голосованием.
14.3. Системные рыночные механики (ордербук, аукцион и т.д.)
      определяются в Правилах.

### Статья 15. Банкротство
15.1. Игрок считается банкротом, когда его суммарные активы
      не покрывают обязательств.
15.2. Банкротство не приводит к удалению из игры.
15.3. Процедура банкротства определяется в Законах.

---

## Раздел VI. Целостность системы

### Статья 16. CI как закон
16.1. Каждый PR обязан пройти CI pipeline.
16.2. CI проверяет: тесты, классификацию уровня, формат, совместимость.
16.3. PR, не прошедший CI, не может быть принят независимо от голосов.

### Статья 16а. Требования к PR с кодом и новыми механиками
16а.1. Любой PR, содержащий новый или изменённый код, должен включать
       тесты с покрытием 100% добавленных/изменённых строк.
16а.2. CI автоматически проверяет покрытие. PR с покрытием ниже 100%
       не проходит CI и не может быть принят.
16а.3. Любой PR, вводящий новую механику или изменяющий существующую,
       должен содержать описание в формате Use Case или User Story.
16а.4. Use Case/User Story размещается в директории `docs/use-cases/`
       и должен включать: цель, участников, сценарий, ожидаемый результат.
16а.5. PR без описания Use Case/User Story для новых механик
       не проходит CI и не может быть принят.

### Статья 17. Прозрачность
17.1. Все действия в мире логируются.
17.2. Состояние мира доступно любому гражданину в любой момент.
17.3. Исходный код CI, сервера и правил — публичен.

### Статья 18. Защита от злоупотреблений
18.1. Мультиаккаунты запрещены. Обнаружение влечёт лишение всех аккаунтов.
18.2. Эксплойты в коде должны быть зарепорчены, а не использованы.
      Ответственный репорт вознаграждается из казны.
18.3. Использование эксплойта без репорта — основание для санкций,
      определённых в Законах.

---

## Раздел VII. Вступление в силу

### Статья 19. Ратификация
19.1. Настоящая Конституция вступает в силу после одобрения
      первыми 3 (тремя) гражданами-основателями.
19.2. До ратификации действует как временный регламент.

### Статья 20. Поправки к Конституции
20.1. Поправки принимаются по процедуре уровня "Конституция"
      (75%+ граждан, 7 дней cooling, тройной залог).
20.2. Раздел I (Неизменяемые принципы) не подлежит изменению.
20.3. Данная статья (20.2) не подлежит изменению.
