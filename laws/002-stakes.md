# Закон 002. О залогах и вознаграждениях за PR

> Уровень: Закон (60%+ голосовавших, 5 дней cooling, x2 залог)
> Статус: драфт

---

## Раздел 1. Размеры залогов

1.1. Залоги за подачу PR определяются по уровню изменений:

| Уровень | Залог | Множитель |
|---------|-------|-----------|
| Конституция | 150 merits | x3 от базы |
| Законы | 100 merits | x2 от базы |
| Правила | 50 merits | x1 (база) |
| Код | без залога | — |

1.2. Базовый залог (50 merits) может быть изменён голосованием
     на уровне "Законы".

## Раздел 2. Механика залога

2.1. Залог блокируется на счёте игрока в момент открытия PR.
2.2. Заблокированные merits не могут быть использованы для торговли,
     производства или других залогов.
2.3. Если у игрока недостаточно средств для залога — PR не может быть подан.

## Раздел 3. Возврат и вознаграждение

3.1. При принятии PR:
     - Залог возвращается полностью.
     - Автор получает вознаграждение из казны = 50% от залога.
3.2. При отклонении PR:
     - Залог переходит в казну полностью.
3.3. При истечении срока PR без набора голосов:
     - Залог возвращается за вычетом 10% (комиссия в казну).

## Раздел 4. Защита от спама

4.1. Гражданин не может иметь более 3 активных PR одновременно.
4.2. Повторная подача отклонённого PR возможна не ранее чем
     через cooling period соответствующего уровня.
4.3. CI автоматически проверяет соблюдение лимитов.

## Раздел 5. Вознаграждение за bug bounty

5.1. Ответственный репорт эксплойта вознаграждается из казны.
5.2. Размер вознаграждения определяется голосованием на уровне "Правила"
     по каждому конкретному случаю.
5.3. Минимальное вознаграждение — 50 merits.
